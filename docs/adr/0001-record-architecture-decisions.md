# 1. Record architecture decisions

## Status
Accepted

## Context
We need to record the architectural decisions made on this project. This is especially important for a medical AI platform where decisions regarding data privacy, model compliance, and security need to be thoroughly documented and traceable.

## Decision
We will use Architecture Decision Records (ADRs), as described by Michael Nygard. We will store them in the `docs/adr` directory. 

## Consequences
See Michael Nygard's article, linked above. For a successful project, we need to know why decisions were made to avoid second-guessing later. This establishes a culture of deliberate architectural design.
